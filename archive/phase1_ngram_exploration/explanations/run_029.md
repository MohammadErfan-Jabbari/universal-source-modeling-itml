# Run 029 — n=5 fixed/adaptive model mixture

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace05_model_mix.py`. It takes the current best Run 027 model mixture and changes only the maximum order from `n=4` to `n=5`, so it can softly use length-4 contexts. The two component distributions remain:

1. fixed-confidence half-count interpolation with `C=44`;
2. adaptive-diversity half-count interpolation with `C_eff = 4 * T(context)`;

then a 50/50 probability mixture.

## Source and evidence
- source: experiment observation — Run 027 is current best at `2.9771224184`, showing fixed/adaptive model averaging helps.
- source: experiment observation — Earlier n=5 hard-backoff/interpolation runs failed, but they did not have half-count local smoothing plus diversity-adaptive confidence plus model averaging.
- source: experiment observation — Run 028 showed mixture-weight tuning is saturated, so changing context availability is a qualitatively different test rather than another public-practice weight tweak.
- source: course material — `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` says repeated longer past patterns are probability evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixtures; `lectures/5_04022026_Inequalities_study.md` ties log-loss excess to mismatch.

## Course-material connection
Longer contexts can reduce conditional entropy when they are genuinely informative, but they increase estimation variance. This run asks whether the current robust mixture/escape mechanism can harvest longer repeated histories while retaining lower-order mass when length-4 contexts are ambiguous or sparse. It remains strictly sequential: order-4 counts are updated only after each observed symbol.

## Hypothesis
The fixed/adaptive half-count mixture may safely use some length-4 context signal that earlier hard or simpler interpolated n=5 models overtrusted. If length-4 contexts are intrinsically noisy for this source, the extra order will again worsen.

## Risks
This deliberately revisits n=5 after multiple n=5 failures, so the expected risk is high. Runtime will increase but should remain far below 600 seconds. Any improvement would need validation because it expands the model class on public practice.
