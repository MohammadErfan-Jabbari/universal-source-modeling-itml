# Run 052 — n=3 recent component with window 32768 and weight 0.095

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_w095_win32768.py`. It keeps the current best Run 050 architecture and changes only the recent-window length upward:

```text
main model:        n=5, alpha=0.3, train+online counts
recent model:      n=3, alpha=0.3
```

This is the opposite coarse window bracket after the failed shorter-window Run 051.

## Source and evidence
- source: experiment observation — Run 050 is current best but the recent-weight gain over `0.08` is tiny, so avoid more narrow weight tuning.
- source: experiment observation — Run 051 showed window `8192` is too noisy for n=3 recent at weight `0.095`; window `16384` remains best so far.
- source: experiment observation — Run 043 showed `32768` was slightly worse for the earlier n=5 recent model, but n=3 recent has lower variance and higher weight, so one broad-window bracket is still informative.
- source: ideas backlog — online recency-window adaptation is a useful mechanism, with the risk that too much or too little locality can hurt.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess bits to model mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixture distributions under source uncertainty.

## Course-material connection
The recent window length controls the bias-variance tradeoff of the local-history source estimate. A broader window reduces estimation variance but may dilute local regimes. The mixture keeps the predictor sequentially valid and hedges the local model against the cumulative n=5 train+online model.

## Hypothesis
If Run 051 failed because even n=3 recent counts need more samples, then window `32768` may improve by stabilizing the recent distribution. If `16384` already balances locality and variance, `32768` will dilute the local-regime signal and worsen.

## Risks
This is another public-practice recency bracket. A small improvement should be validated rather than followed by more fine window tuning. Runtime and memory may rise slightly but should remain far below the 600-second limit. The official evaluator and data are unchanged and the predictor uses only past symbols.
