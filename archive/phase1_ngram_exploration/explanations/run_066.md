# Run 066 — all-past n=3 auxiliary smoothing alpha 0.05

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_recenta005_w14_warm20000_win262144.py`. It keeps the current best Run 065 architecture and changes only the all-past online component's smoothing:

```text
main model:          n=5, alpha=0.3, train+online counts
online component:    n=3, alpha=0.05, test-prefix-only counts
window:              262144 observed symbols (> official 200000-token prefix)
max mixture weight:  0.14
warmup:              20000 tokens
```

The main high-order model remains at `alpha=0.3`; this is only a lower-side bracket for the dense low-order auxiliary component.

## Source and evidence
- source: experiment observation — Run 064 improved with auxiliary `alpha=0.2`, and Run 065 improved again with auxiliary `alpha=0.1`, so the low-order all-past online component was still over-smoothed at `0.2`.
- source: experiment observation — Runs 052-055 showed full-prefix history stabilizes the auxiliary `n=3` counts, making sharper smoothing plausible.
- source: experiment observation — Run 033 showed globally sharp smoothing can hurt sparse `n=5` contexts; therefore this run leaves the main model unchanged and only sharpens the denser auxiliary model.
- source: official/project rules — `AGENTS.md` and `COMPETITION_RULES.md` require online prediction and fix `N=200000`; the predictor uses only past symbols and preserves evaluator semantics.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess bits to cross-entropy/KL mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixtures that hedge empirical source estimates.

## Course-material connection
Lowering auxiliary smoothing trusts the empirical test-prefix history more. This can reduce cross-entropy if the low-order online empirical distribution is well estimated, but it risks redundancy from rare-symbol surprise penalties. Because the final predictor is a convex mixture with the main model, it remains a valid sequential coding distribution.

## Hypothesis
If auxiliary `alpha=0.1` is still over-smoothed, `alpha=0.05` will improve over Run 065 (`2.9718452410`). If `0.1` is already near the rare-symbol protection limit, `0.05` will be too sharp and worsen.

## Risks
This is an aggressive smoothing reduction. It may assign too little mass to unseen symbols in low-count contexts, especially before the warmup finishes. Runtime and memory should be unchanged. Any small gain should be validated because this is public-practice calibration.
