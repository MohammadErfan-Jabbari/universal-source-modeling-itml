# Run 032 — n=5 model mixture with additive smoothing 0.3

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_model_mix.py`. It keeps the current best Run 031 structure (`n=5`, fixed/adaptive confidence schedules, 50/50 model mixture), but lowers local additive smoothing from `0.4` to `0.3`.

## Source and evidence
- source: experiment observation — Run 031 improved substantially to `2.9754042414` by lowering n=5 model-mixture smoothing from `0.5` to `0.4`.
- source: experiment observation — Run 030 showed n=6 is too sparse/noisy, so tuning should stay at n=5.
- source: experiment observation — In n=4, too-low smoothing eventually hurt, so this is a coarse bracket rather than a fine search.
- source: course material — `lectures/5_04022026_Inequalities_study.md` says cross-entropy excess is model mismatch; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixtures of source models.

## Course-material connection
The model mixture supplies lower-order and alternative-confidence robustness. Reducing the pseudocount sharpens the local context-source models, which can lower code length if those contexts are informative. The mixture prevents zero probabilities and keeps the predictor sequentially valid.

## Hypothesis
If Run 031 improved because n=5 contexts are still over-smoothed at 0.4, then 0.3 will reduce mismatch further. If 0.4 is near the bias-variance optimum, 0.3 will underweight surprising symbols in sparse order-4 contexts and worsen.

## Risks
This is another public-practice hyperparameter bracket, so the result should be validated before trust. Runtime should be similar to Run 031 and far below the timeout. No future test symbols are used.

Run:

