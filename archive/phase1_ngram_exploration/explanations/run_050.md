Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_w095_win16384.py`. It keeps the current best Run 048 architecture and changes only the maximum recent-window mixture weight:

- source: experiment observation — Run 049 worsened at n=3 recent max weight `0.11`, but was still better than weight `0.05`, bracketing the useful range between about `0.08` and `0.11`.
- source: experiment observation — Run 047 worsened with recent `n=2`, so keep the recent order fixed at `n=3` for this single weight bracket.
- source: ideas backlog — recency-window adaptation is useful but can chase noise if overweighted.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties score excess to cross-entropy/KL mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as evidence about repeated structure; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates mixture distributions as robust codes under source uncertainty.

## Course-material connection
The recent mixture weight trades stable global evidence against local-history evidence. In coding terms, it decides how much probability mass to reserve for a local source model. If the local source is closer to the current stream regime, a moderate weight increase reduces redundancy; if not, it adds mismatch.

## Hypothesis
If `0.08` underweights reliable n=3 recent evidence but `0.11` is too aggressive, then `0.095` may improve. If `0.08` is already at or near the optimum, this moderate increase will still dilute main-model information and worsen.

## Risks
This is a fine public-practice bracket, so it should be treated as exploratory and not repeated indefinitely. A small improvement needs validation on train-derived splits or repeat runs. Runtime should match Run 048 and remain far below timeout. No evaluator, test data, or future symbols are used.
