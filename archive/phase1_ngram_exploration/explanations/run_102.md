# Run 102 — per-order fixed_c interpolation

## Proposed change
Add `FIXED_C_BY_ORDER = (30, 30, 50, 70, 90)`. Each context order gets its own interpolation constant, complementing per-order smoothing:

```text
main model:   n=5
  order 0: alpha=0.02, C=30 (very dense, trust local estimate)
  order 1: alpha=0.02, C=30 (very dense)
  order 2: alpha=0.05, C=50 (moderate)
  order 3: alpha=0.15, C=70 (getting sparse)
  order 4: alpha=0.3,  C=90 (sparse, borrow more from lower orders)

aux: n=3, alpha=0.01, pure fixed-C, weight 0.10, warmup 15000
```

## Source and evidence
- Runs 083-085: uniform C optimum at 60 with uniform alpha=0.3
- Runs 099-101: uniform C optimum shifted to 70 with per-order alpha
- The pattern: sharper alpha → higher C needed → but this interaction varies by order. Per-order C lets each depth find its own balance.

## Course-material connection
Interpolation constant trades variance (trusting local counts) vs bias (borrowing lower-order statistics). Dense contexts can afford lower C (trust sooner); sparse contexts need higher C (more conservative). Per-order smoothing and interpolation naturally pair: both are about density-dependent calibration.

## Hypothesis
If dense low-order contexts benefit from lower C and sparse high-order from higher C simultaneously, per-order C improves over uniform C=70 (`2.9700868382`).

## Risks
Per-order C interacts with per-order alpha; the chosen values are educated guesses. Small calibration.
