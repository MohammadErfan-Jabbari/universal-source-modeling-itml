# Run 013 — interpolated n=4 with C=40

## Proposed change
Add `submissions/ngram_interpolated_n4_c40.py`, another member of the recursive interpolated n-gram family. It uses interpolation constant `C=40`, between the former best `C=32` and current best `C=48`.

## Source and evidence
- source: experiment observation — Run 010 (`C=32`) scored `2.9794649970`; Run 012 (`C=48`) improved slightly to `2.9792935749`; Run 011 (`C=64`) worsened to `2.9798657316`.
- source: experiment observation — The best region appears near 32–48 rather than at 64, but the effect is tiny and could be noise.
- source: ideas backlog — `autoresearch.ideas.md` proposes count-dependent mixture weights `count/(count+c)` for robust context-order mixtures.
- source: course material — `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates mixture distributions as robust universal codes; `lectures/5_04022026_Inequalities_study.md` frames the score gap as mismatch `D(P||Q)`.

## Course-material connection
Changing `C` changes the mixture over context-order models. `C=40` is a local search step to balance the mismatch from overtrusting order-3 contexts against the mismatch from oversmoothing toward lower orders. The model remains strictly sequential and updates only after each observed symbol.

## Hypothesis
If the optimum is between C=32 and C=48, `C=40` may beat both. If the apparent C=48 gain is noise or the optimum is closer to 48, `C=40` will be slightly worse.

## Risks
The expected difference is extremely small, below current confidence estimates, so public-practice tuning may overfit. Runtime and probability validity should be essentially unchanged from the other interpolated n=4 variants.

Run:

```bash
PREDICTOR_PATH=submissions/ngram_interpolated_n4_c40.py ./autoresearch.sh
```

Success means `bits_per_symbol < 2.9792935749` for a new best, with `timed_out=0`, `evaluated_tokens=200000`, and automatic checks passing.
