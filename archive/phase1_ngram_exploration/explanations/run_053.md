Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_w095_win65536.py`. It keeps the current best Run 052 architecture and changes only the recent-window length upward again:


- source: experiment observation — Run 050 showed recent-weight tuning was flat/noise-sensitive, so window length is the more meaningful active dimension.
- source: ideas backlog — recency-window adaptation can capture local structure, but the window must balance locality and sample size.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess bits to model mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixture distributions under source uncertainty.

## Course-material connection
Window length changes the finite-history evidence used by the local model. A larger window gives lower-variance estimates for n=3 contexts, but at some point it becomes less local and starts resembling the cumulative model. The mixture with the n=5 main model remains a valid sequential coding distribution.

## Hypothesis
If the recent n=3 model is still sample-limited at window `32768`, then `65536` will improve by further stabilizing local probabilities. If `32768` already captures the right locality/variance balance, `65536` will dilute useful local regimes and worsen.

## Risks
This continues public-practice recency tuning, so a small improvement should be validated or repeated. Runtime and memory will rise, but should remain far below the 600-second limit. The predictor still uses only past symbols, no external calls, and no evaluator changes.
