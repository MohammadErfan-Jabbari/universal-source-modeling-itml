# Run 069 — all-past n=3 auxiliary alpha 0.01 with weight 0.16

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_recenta001_w16_warm20000_win262144.py`. It keeps the current best Run 068 auxiliary smoothing and warmup, but retunes the mixture weight upward:

```text
main model:          n=5, alpha=0.3, train+online counts
online component:    n=3, alpha=0.01, test-prefix-only counts
window:              262144 observed symbols (> official 200000-token prefix)
max mixture weight:  0.16 instead of 0.14
warmup:              20000 tokens
```

This is not another lower-smoothing run; it asks whether the now-sharper auxiliary component deserves slightly more mixture mass.

## Source and evidence
- source: experiment observation — Runs 064-068 improved by sharpening only the all-past `n=3` auxiliary model, but Run 068's gain at `alpha=0.01` was tiny, so further aggressive smoothing is not the next best direction.
- source: experiment observation — The last weight tuning was done before auxiliary smoothing was separated; if the auxiliary distribution is now better calibrated, its optimal mixture weight may change.
- source: experiment observation — Run 058 showed weight `0.18` was too high for the older `alpha=0.3` auxiliary model, so `0.16` is a cautious upper retune rather than a large jump.
- source: official/project rules — `AGENTS.md` and `COMPETITION_RULES.md` require online prediction and fix `N=200000`; this remains a convex mixture of distributions computed only from past symbols.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess bits to cross-entropy/KL mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixtures over source estimates.

## Course-material connection
The weight is the universal-coding hedge between the train-heavy longer-context model and the test-prefix-specific low-order model. After improving the auxiliary probability estimate, the old hedge may underallocate mass to it. Increasing weight helps only if the sharper auxiliary reduces mismatch more often than it amplifies its mistakes.

## Hypothesis
If the sharpened `alpha=0.01` all-past `n=3` component is now underweighted, max weight `0.16` will improve over Run 068 (`2.9718045659`). If the component is already near its best mixture mass, `0.16` will dilute the main model and worsen.

## Risks
This is public-practice mixture retuning after several smoothing runs, so any small gain should be validated. A higher weight can amplify rare-symbol penalties from the sharp auxiliary model. Runtime and memory should remain unchanged.
