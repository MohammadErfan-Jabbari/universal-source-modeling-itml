# Run 018 — interpolated n=4 C=44 with KT-style smoothing

## Proposed change
Add `submissions/ngram_interpolated_n4_c44_laplace05.py`. It keeps the current best model structure (`n=4`, recursive count-dependent interpolation, `C=44`, unit online adaptation) but changes the local additive smoothing from Laplace `1.0` to `0.5`, the Krichevsky-Trofimov/Jeffreys-prior style pseudocount for a finite alphabet.

## Source and evidence
- source: experiment observation — Runs 009-015 show n=4 recursive interpolation is the successful branch, with the C optimum flat around 40-44.
- source: experiment observation — Run 016 showed order-4 contexts hurt even under interpolation, and Run 017 showed overweighting online updates hurts; keep the model structure stable and tune the local estimator instead.
- source: experiment observation — Run 005 showed hard-backoff n=4 with smoothing `0.5` was only slightly worse than Laplace `1.0`; interpolation may supply extra lower-order safety and make sharper local estimates beneficial.
- source: agent prior knowledge — KT/Jeffreys smoothing uses a half-count prior as a robust Bayesian universal-coding estimator for discrete alphabets.
- source: course material — `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates mixture distributions for universal coding; `lectures/5_04022026_Inequalities_study.md` says cross-entropy excess is exactly mismatch cost.

## Course-material connection
The current predictor already mixes context orders, but each local context distribution still uses a symmetric prior. Changing the pseudocount changes how much probability mass is reserved for symbols not yet frequent in that context. A half-count prior is a less conservative universal estimate than add-one smoothing, aiming to reduce redundancy when context counts are moderately informative while keeping every symbol positive.

## Hypothesis
In the interpolated model, lower-order mixtures already protect against rare surprises, so local order-2/3 estimates may benefit from sharper KT-style smoothing. This could reduce mismatch in supported contexts. If add-one smoothing was providing necessary robustness, `0.5` will underweight surprising symbols and worsen as it did for hard backoff.

## Risks
The improvement may be tiny or public-practice-specific. Lower smoothing can make rare symbols too unlikely inside each local distribution, though interpolation and positive pseudocounts preserve validity. Runtime should be essentially unchanged from the C=44 interpolated model.
