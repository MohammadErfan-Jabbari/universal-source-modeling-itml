# Run 044 — n=5 main model with n=4 recent-window component

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n4_win16384.py`. It keeps the current best Run 041 cumulative main model and recent-mixture settings:

```text
main model:   n=5, alpha=0.3, fixed/adaptive mixture
recent model: n=4, alpha=0.3, fixed/adaptive mixture
window:       16384 observed symbols
max weight:   0.05 with 10000-token ramp
```

Only the online-only recent component drops its maximum context length from order 4 to order 3.

## Source and evidence
- source: experiment observation — Run 041 found the best recent-window setting so far: window `16384`, weight `0.05`, score `2.9739917554`.
- source: experiment observation — Run 042 showed window `4096` is too noisy, and Run 043 showed window `32768` is too broad; the window is bracketed.
- source: experiment observation — Earlier n=5 hard models failed from sparse long contexts, while robust n=5 cumulative mixtures worked. The recent table has fewer samples than the cumulative model, so a lower recent order may reduce variance.
- source: course material — `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` says repeated past history is evidence, but finite samples matter; `lectures/5_04022026_Inequalities_study.md` ties log-loss to mismatch; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixture coding distributions.

## Course-material connection
This is a bias-variance adjustment inside the recent source model. The cumulative model can afford n=5 because it has training counts plus all past test counts. The recent model only sees a sliding window, so using n=4 may provide a more reliable local distribution while the main n=5 model still captures longer repeated histories.

## Hypothesis
If the recent n=5 component is variance-limited even at window `16384`, then an n=4 recent component will improve. If the useful recency signal lives specifically in length-4 contexts, lowering recent order will discard it and worsen.

## Risks
This changes model structure after several recency brackets, so small improvements need validation. Runtime should be similar or slightly faster than Run 041. The predictor remains strictly sequential: recent counts are updated after each observed symbol and old counts are removed from the window.
