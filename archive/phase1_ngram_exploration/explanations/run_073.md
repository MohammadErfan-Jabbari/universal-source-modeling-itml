# Run 073 — separate recent fixed/adaptive mixture weight 0.75

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_recenta001_w16_warm30000_mix075_wall262144.py`. It keeps the current best Run 071 architecture but gives the all-past `n=3` auxiliary model its own internal fixed/adaptive mixture weight:

```text
main model:             n=5, alpha=0.3, mix_weight_fixed=0.5 (unchanged)
auxiliary component:    n=3, alpha=0.01, recent_mix_weight_fixed=0.75
window:                 262144 observed symbols (> official 200000-token prefix)
max mixture weight:     0.16
warmup:                 30000 tokens (linear)
```

Previously both the main model and the auxiliary model used the same `mix_weight_fixed=0.5` to blend a fixed-C44 and adaptive-C (diversity-dependent) version of each n-gram estimate. This run trusts the simpler fixed-C44 interpolation more heavily for the auxiliary low-order model, which has dense counts from the full past prefix.

## Source and evidence
- source: experiment observation — Run 027 originally introduced the 50/50 fixed/adaptive mixture, finding it improved over either fixed or adaptive alone for the full n=4 model. That was for a model covering all orders including sparse high-order contexts.
- source: experiment observation — The all-past n=3 auxiliary component has dense low-order counts accumulated from essentially all observed test symbols (Runs 052-055 showed it stabilizes with full-prefix history). The adaptive diversity-based C_eff may be less necessary or even slightly mis-calibrated for dense low-order contexts.
- source: experiment observation — Runs 064-068 sharpened the auxiliary smoothing enough that the main model and auxiliary model are now qualitatively different distributions. It's plausible they differ in their optimal internal mixture as well.
- source: official/project rules — `AGENTS.md` and `COMPETITION_RULES.md` require online prediction and fix `N=200000`; this change preserves sequential validity, the predictor remains a convex mixture of valid distributions.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess bits to cross-entropy/KL mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixtures over source estimates.

## Course-material connection
The fixed-C and adaptive-C estimators are two different source estimates of the same empirical context distribution. Weighting them is a universal-coding hedge. For dense low-order contexts the simpler fixed-C calibration may be more reliable; for sparse high-order contexts the adaptive diversity-based C_eff provides important protection. Decoupling the two allows each model to use the right balance.

## Hypothesis
If the all-past n=3 auxiliary contexts are dense enough that fixed-C44 is better calibrated than the 50/50 blend, `recent_mix_weight_fixed=0.75` will improve over Run 071 (`2.9717926651`). If the 50/50 blend is still the better hedge, the over-weighted fixed-C component will worsen.

## Risks
This is a small calibration change. Runtime and memory should be essentially unchanged. Any gain should be validated on a train-derived split before strong claims.
