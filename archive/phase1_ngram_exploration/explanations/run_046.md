# Run 046 — n=5 main model with n=3 recent-window component

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_win16384.py`. It keeps the current best Run 044 setup, but lowers only the recent-window component one more order:

```text
main model:   n=5, alpha=0.3, train+online counts
recent model: n=3, alpha=0.3, window=16384, max weight=0.05
```

The cumulative main model still uses n=5; only the online-only recent model uses shorter contexts.

## Source and evidence
- source: experiment observation — Run 044 improved by lowering the recent component from n=5 to n=4, indicating the sliding recent table is variance-limited.
- source: experiment observation — Run 045 showed increasing recent smoothing to `0.5` worsened, so keep recent `alpha=0.3` and bracket order instead.
- source: experiment observation — Earlier n=5 failures and Run 030's n=6 failure show sparse long contexts are dangerous without enough data.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess log-loss to mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats past recurrence as evidence but finite-window evidence has variance; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixtures over source assumptions.

## Course-material connection
A shorter recent context is a stronger bias-variance tradeoff: it ignores some local history but estimates the remaining local distribution with many more samples. The main n=5 model still carries longer-memory evidence from the full training/cumulative history, so the recent component can focus on reliable medium-short local adaptation.

## Hypothesis
If the recent n=4 model is still too sparse, recent n=3 will improve by further reducing variance. If useful recent structure depends on length-3 suffixes (n=4 model contexts), lowering to n=3 will discard important local signal and worsen.

## Risks
This is public-practice recency tuning and should be validated if it improves. Runtime should be similar or slightly faster than Run 044. The predictor remains strictly sequential: recent counts update only after observing each symbol.
