# Run 064 — all-past n=3 auxiliary smoothing alpha 0.2

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_recenta02_w14_warm20000_win262144.py`. It keeps the current best Run 061 mixture weight and warmup, but separates the smoothing used by the all-past online component:

```text
main model:          n=5, alpha=0.3, train+online counts
online component:    n=3, alpha=0.2, test-prefix-only counts
window:              262144 observed symbols (> official 200000-token prefix)
max mixture weight:  0.14
warmup:              20000 tokens
```

Only the auxiliary component's Lidstone/additive smoothing changes; the train+online main model remains at the known-good `alpha=0.3`.

## Source and evidence
- source: experiment observation — Run 061 is current best with all-past `n=3`, weight `0.14`, and warmup `20000`.
- source: experiment observation — Runs 062-063 bracketed warmup and showed narrow warmup tuning is now flat or worse, so a qualitative calibration dimension is preferable.
- source: experiment observation — Runs 052-055 showed the all-past auxiliary component is sample-limited but becomes stable with full-prefix history; a stable low-order online component may tolerate sharper smoothing than the sparse main `n=5` model.
- source: experiment observation — Run 033 showed global `alpha=0.2` is too sharp for the full `n=5` main mixture, which is why this run changes only the all-past `n=3` auxiliary component.
- source: official/project rules — `AGENTS.md` and `COMPETITION_RULES.md` require online prediction and fix `N=200000`; separate smoothing uses only past counts and does not alter evaluator semantics.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess log-loss to model mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates mixtures that hedge different source estimates.

## Course-material connection
Additive smoothing controls the bias-variance tradeoff of a probability estimate. For the all-past low-order online model, counts are much denser than for the main high-order model, so less smoothing may reduce mismatch without creating too many surprise penalties. The final distribution remains a convex mixture of valid sequential distributions.

## Hypothesis
If full-prefix `n=3` online contexts are dense enough, auxiliary `alpha=0.2` will sharpen the test-specific low-order probabilities and improve over Run 061 (`2.9719344575`). If rare symbols still need the `0.3` pseudocount protection, the sharper auxiliary component will worsen.

## Risks
Lower smoothing can over-penalize symbols not yet seen in a context, especially early in the stream, though the 20000-token warmup mitigates this. This is a public-practice calibration run and should be validated if it gives only a tiny gain. Runtime and memory should be essentially unchanged.
