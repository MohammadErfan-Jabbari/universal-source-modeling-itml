# Run 009 — count-dependent interpolated n=4 n-gram

## Proposed change
Add `submissions/ngram_interpolated_n4_c16.py`. It keeps `n=4`, Laplace `1.0`, and online adaptation, but replaces hard backoff prediction with recursive interpolation over orders 0..3. For order `k>0`, the local estimate is mixed with the shorter-order mixture using confidence `lambda_k = count_k / (count_k + 16)`.

## Source and evidence
- source: experiment observation — Runs 002-003 and 007-008 showed higher-order hard/thresholded `n=5` contexts are harmful; Run 005 showed baseline `n=4`, Laplace `1.0` remains best among smoothing variants; Run 006 showed online adaptation should stay enabled.
- source: ideas backlog — `autoresearch.ideas.md` proposes mixtures of context orders and count-dependent weights so rare contexts borrow lower-order statistics.
- source: course material — `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` says the optimal universal coding distribution is a mixture under a prior, and that the best average-case code for a prior is a mixture distribution.
- source: course material — `lectures/5_04022026_Inequalities_study.md` gives `H(P,Q)=H(P)+D(P||Q)`, so interpolation is an attempt to reduce conditional model mismatch without committing to one context order.

## Course-material connection
Hard backoff behaves like choosing one source model/order and coding entirely with it. Interpolation behaves like a small universal-model mixture: when a high-order context has many observations, it gets high weight; when it is uncertain, shorter contexts still carry probability mass. This is a sequential, no-lookahead model average over Markov orders.

## Hypothesis
The baseline may overcommit to order-3 conditional estimates even when they are moderately noisy. A count-dependent mixture with `C=16` should stay close to the baseline in well-supported contexts while improving calibration in weaker contexts by borrowing order-0/1/2 evidence.

## Risks
The interpolation constant may mix too much lower-order mass and blur useful order-3 signal, raising log-loss. More per-token computation may slow evaluation, though it is still only four 16-symbol distributions per step. Any small public-practice gain would need later train-derived validation.

## Validation plan
