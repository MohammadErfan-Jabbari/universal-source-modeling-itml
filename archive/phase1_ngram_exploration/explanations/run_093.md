# Run 093 — per-order smoothing for main model

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_perorder_recent_n3_recenta001_w10_warm30000_mix1_main085_FIXEDC60_wall262144.py`. Main model uses sharper smoothing for dense low-order contexts while keeping alpha=0.3 only for sparse high-order slices:

```text
main model:             n=5
  order 0: alpha=0.1   (unconditional, very dense)
  order 1: alpha=0.1   (single-symbol context, very dense)
  order 2: alpha=0.15  (moderately dense)
  order 3: alpha=0.25  (getting sparse)
  order 4: alpha=0.3   (n=5 high-order, sparse, needs protection)
  mix_weight_fixed=0.85, fixed_c=60

auxiliary:              n=3, alpha=0.01, pure fixed-C, weight 0.10
warmup:                 30000
```

Only the main model smoothing schedule changes. The auxiliary remains at alpha=0.01.

## Source and evidence
- source: experiment observation — Runs 064-068 showed the dense low-order auxiliary benefits from very sharp smoothing (alpha=0.01). The main model's lower orders are also dense (accumulated from train + online) and may benefit from sharper smoothing than the shared 0.3.
- source: experiment observation — Run 036 tried order-dependent smoothing with the old simpler architecture (varying only the highest order), but the model has since improved significantly.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess bits to cross-entropy mismatch; lower smoothing reduces bias in dense contexts at the risk of underweighting surprises in sparse contexts.

## Course-material connection
Smoothing is a bias-variance tradeoff per context depth. Dense counts (low orders) can afford sharper estimates; sparse counts (high orders) need conservative pseudocounts to avoid catastrophic surprise penalties. Separating the smoothing schedule acknowledges this structure rather than averaging across it.

## Hypothesis
If lower-order main model contexts are dense enough for sharper smoothing, per-order alphas will improve over uniform alpha=0.3 (`2.9704122304`). If the uniform 0.3 is already well-calibrated across all orders, this will worsen or be flat.

## Risks
The chosen alpha brackets are educated guesses, not tuned. Runtime should be similar. Gains may be small since this affects only the 85%-weighted main model, not the auxiliary.
