# Run 048 — n=3 recent-window component with max weight 0.08

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_w08_win16384.py`. It keeps the current best Run 046 architecture and changes only the maximum recent-window mixture weight:

```text
main model:        n=5, alpha=0.3, train+online counts
recent model:      n=3, alpha=0.3, window=16384
max recent weight: 0.08 instead of 0.05, with the same 10000-token ramp
```

The recent model remains online-only and is updated only after each symbol is observed.

## Source and evidence
- source: experiment observation — Run 046 improved to `2.9734028461` by using recent `n=3`, suggesting the recent component is now less sparse and more reliable than the earlier n=5/n=4 recent variants.
- source: experiment observation — Run 047 worsened with recent `n=2`, bracketing the useful recent order around `n=3`.
- source: experiment observation — Run 039 showed max weight `0.08` worsened for the sparse n=5 recent model, so this is not a generic weight increase; it tests whether the lower-variance n=3 recent model can safely carry more probability mass.
- source: ideas backlog — online recency-window adaptation is a planned mechanism, with the explicit risk that too much recent mass can chase noise.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess log-loss to cross-entropy mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates mixtures as robust distributions under source uncertainty.

## Course-material connection
The mixture weight controls how much codelength risk is allocated to local-history evidence versus the global train+online model. If the recent n=3 source is a better approximation to the current local regime, increasing its weight reduces model mismatch. If it is only a noisy local estimator, the extra mixture mass increases redundancy.

## Hypothesis
Because recent `n=3` is less sparse than previous recent models and beat both recent `n=4` and `n=2`, raising max recent weight from `0.05` to `0.08` may improve by using the local-regime signal more strongly. If the 5% mixture was already near optimal, 8% will overfit recency noise or dilute useful long-context main-model information and worsen.

## Risks
This is another public-practice recency bracket, so treat any small improvement cautiously and validate later. Prior n=5 weight `0.08` failed, so the experiment can easily worsen. Runtime should be similar to Run 046 and safely below timeout. The official evaluator and data remain unchanged.
