# Run 012 — interpolated n=4 with intermediate confidence (C=48)

## Proposed change
Add `submissions/ngram_interpolated_n4_c48.py`, the same recursive interpolated n-gram family as Runs 009-011 but with interpolation constant `C=48`. This brackets the current best `C=32` against the slightly worse `C=64`.

## Source and evidence
- source: experiment observation — Run 010 (`C=32`) is current best at `2.9794649970`; Run 011 (`C=64`) was slightly worse at `2.9798657316` but still better than baseline.
- source: experiment observation — Runs 009-011 show the interpolation branch is promising and the optimum likely lies near, but not necessarily exactly at, `C=32`.
- source: ideas backlog — `autoresearch.ideas.md` proposes count-dependent mixture weights `count/(count+c)` as a model-order mixture.
- source: course material — `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` frames robust universal coding distributions as mixtures; `lectures/5_04022026_Inequalities_study.md` states cross-entropy excess is mismatch cost.

## Course-material connection
`C` controls the model-order mixture prior in operational terms: larger `C` keeps more mass on lower-order contexts unless high-order evidence is strong. Testing `C=48` refines the mixture tradeoff between model specificity and robustness, aiming to reduce empirical code length by lowering conditional mismatch.

## Hypothesis
If the public-practice optimum is between `C=32` and `C=64`, then `C=48` should beat or nearly match `C=32`. If `C=32` is already the best balance, `C=48` will blur useful higher-order information and worsen slightly.

## Risks
The expected effect is small and may be within run noise, so a tiny gain should not be over-interpreted without repeat runs or train-derived validation. Runtime should remain around 3.5–3.7 seconds and probabilities should stay valid as a mixture of normalized positive distributions.
