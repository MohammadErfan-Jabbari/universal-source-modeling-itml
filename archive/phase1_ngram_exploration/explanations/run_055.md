# Run 055 — all-past n=3 online component with weight 0.095

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_w095_win262144.py`. It keeps the current best Run 054 architecture but sets the online-only n=3 component's window above the official evaluated prefix:

```text
main model:        n=5, alpha=0.3, train+online counts
online component:  n=3, alpha=0.3, test-prefix-only counts
max mixture weight: 0.095 with 10000-token ramp
window:            262144 observed symbols (> official 200000-token prefix)
```

Operationally, this makes the former recent component an all-past test-prefix n=3 model for the official 200000-token evaluation, while remaining strictly online.

## Source and evidence
- source: experiment observation — Run 052 improved when broadening the n=3 online window to `32768`.
- source: experiment observation — Run 053 improved again at `65536`, and Run 054 improved again at `131072`, though with shrinking marginal gain.
- source: experiment observation — Run 051 showed shorter window `8192` is too noisy, so the active signal is stable low-order online adaptation rather than short-regime locality.
- source: official rules — `COMPETITION_RULES.md` fixes the evaluated prefix at `N=200000`, so a `262144` cap is effectively all-past for the official run without using future symbols.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess log-loss to model mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixture distributions under source uncertainty.

## Course-material connection
The repeated broad-window gains suggest the auxiliary n=3 component is not merely tracking short local regimes; it is estimating a test-realization-specific low-order source that complements the train-heavy n=5 main model. This is still universal/sequential coding: at each time, the auxiliary model uses only the already observed prefix, and the final distribution is a convex mixture.

## Hypothesis
If the n=3 auxiliary model is still sample-limited at window `131072`, then using all past test-prefix symbols will improve by further stabilizing its low-order probabilities. If recency still matters or the auxiliary model becomes too similar to the main train+online model, the all-past window will dilute useful locality and worsen.

## Risks
The window is deliberately larger than the official prefix, so the model is less a recency model and more a test-only online low-order component. This is allowed but should be explained honestly. It may overfit public-practice prefix length, so improvements need validation. Runtime and memory should remain far below the 600-second limit.
