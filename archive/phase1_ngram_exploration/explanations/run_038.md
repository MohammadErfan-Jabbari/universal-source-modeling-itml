# Run 038 — recent-window online mixture retry without dataclasses

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_mix.py`. This retries the Run 037 modeling idea but replaces the local dataclass count container with a plain manual class so the official importlib loader can import the predictor.

The predictor keeps the current best Run 032 distribution and mixes in a small online-only recent-window distribution:

```text
q = (1 - w_t) q_train+online + w_t q_recent_window
w_t = 0.05 * min(1, past_tokens / 10000)
window = 8192 observed symbols
```

Both components use n=5 fixed/adaptive interpolation and `alpha=0.3`. The recent component starts empty, updates only after each observed symbol, and removes counts older than the window.

## Source and evidence
- source: experiment observation — Run 032 global `alpha=0.3` remains best; Runs 033-036 show smoothing tweaks are saturated or harmful.
- source: experiment observation — Run 037 crashed before testing the idea due to a dataclass/importlib issue, so retrying with a manual class tests the intended model rather than a new hypothesis.
- source: experiment observation — Run 006 showed removing online adaptation hurts, while Run 017 showed simple global online overweighting hurts. A small separate recent model is a different adaptation mechanism.
- source: ideas backlog — `autoresearch.ideas.md` suggests recency-window adaptation and static/online mixtures.
- source: course material — `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` says recent repeated history can reveal structure without knowing the source law; `lectures/5_04022026_Inequalities_study.md` ties extra code length to model mismatch; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates mixture distributions for robust universal coding.

## Course-material connection
The main cumulative model estimates a stable conditional law from training plus all past test tokens. A recent-window model is a controlled hedge for local regimes or realization-specific recurrence. Mixing the two is still a valid sequential coding distribution because both components use only past observations.

## Hypothesis
If the evaluated stream has local state or recency effects not captured by cumulative counts, a 5% ramped recent-window mixture will improve log-loss. If the source is effectively stationary at this scale, the recent model will mostly add noisy sparse estimates and worsen.

## Risks
The recent component can overfit short-term noise, especially in order-4 contexts. The small ramped weight is meant to limit damage. Runtime will increase due to maintaining and scoring two count tables but should remain far below 600 seconds.
