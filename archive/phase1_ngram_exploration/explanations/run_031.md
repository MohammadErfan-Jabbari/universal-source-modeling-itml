# Run 031 — n=5 model mixture with additive smoothing 0.4

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace04_model_mix.py`. It keeps the current best Run 029 structure (`n=5`, fixed/adaptive confidence schedules, 50/50 model mixture), but lowers local additive smoothing from `0.5` to `0.4`.

## Source and evidence
- source: experiment observation — Run 029 is current best at `2.9766819653` with `n=5`, half-count smoothing, and fixed/adaptive model averaging.
- source: experiment observation — Run 030 showed n=6 is too sparse/noisy, so further order increases should stop.
- source: experiment observation — In the n=4 family, `0.5` beat both `0.25` and `0.75`, but n=5 adds another sparse high-order component that may require slightly different local smoothing.
- source: course material — `lectures/5_04022026_Inequalities_study.md` says every cross-entropy excess bit is mismatch; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixtures of candidate source models.

## Course-material connection
The n=5 mixture is a sequential universal-style model average. The additive pseudocount is the local prior inside each context-order source model. Lowering it slightly asks whether the robust mixture already supplies enough backoff protection that local contexts can be a bit sharper without underweighting surprises.

## Hypothesis
If the n=5 mixture's high-order contexts are informative and the model average protects rare symbols, smoothing `0.4` may reduce local mismatch versus `0.5`. If the half-count prior is already the right robustness point, `0.4` will overtrust sparse high-order counts and worsen.

## Risks
This is a narrow hyperparameter bracket around the current best. If it fails or only ties, stop smoothing retuning and validate/repeat the n=5 best. The method remains strictly sequential and runtime should be similar to Run 029.
