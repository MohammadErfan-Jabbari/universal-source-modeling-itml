# Run 065 — all-past n=3 auxiliary smoothing alpha 0.1

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_recenta01_w14_warm20000_win262144.py`. It keeps the Run 064 architecture and changes only the all-past online component's smoothing:

```text
main model:          n=5, alpha=0.3, train+online counts
online component:    n=3, alpha=0.1, test-prefix-only counts
window:              262144 observed symbols (> official 200000-token prefix)
max mixture weight:  0.14
warmup:              20000 tokens
```

The main model, auxiliary order, weight, and warmup stay fixed; this is a lower-side smoothing bracket after auxiliary `alpha=0.2` improved.

## Source and evidence
- source: experiment observation — Run 064 improved to `2.9718899997` by lowering only the all-past `n=3` auxiliary smoothing from `0.3` to `0.2`.
- source: experiment observation — Runs 052-055 showed the auxiliary component becomes stable with essentially all past test-prefix symbols, so it may tolerate sharper smoothing than sparse high-order models.
- source: experiment observation — Run 033 showed global `alpha=0.2` was too sharp for the full `n=5` main model, so this bracket is deliberately restricted to the denser low-order online component.
- source: official/project rules — `AGENTS.md` and `COMPETITION_RULES.md` require online prediction and fix `N=200000`; the auxiliary probabilities use only past counts and preserve official evaluator semantics.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess bits to cross-entropy/KL mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates hedging different source estimates in a mixture.

## Course-material connection
Smoothing is a bias-variance choice for the low-order online distribution. Less smoothing trusts the empirical synchronized history more, which can reduce mismatch when contexts are dense. But too little smoothing creates high surprise penalties for symbols not yet seen in a context. The final distribution remains a valid sequential convex mixture.

## Hypothesis
If the all-past `n=3` online contexts are dense enough that `alpha=0.2` is still over-smoothed, `alpha=0.1` will improve over Run 064 (`2.9718899997`). If `alpha=0.2` is near the best calibration, `0.1` will over-sharpen and worsen.

## Risks
This lower smoothing can underweight rare/unseen symbols, especially early, although the 20000-token warmup and main-model mixture limit damage. This is still public-practice hyperparameter bracketing, so a small gain should be validated. Runtime and memory should be unchanged.
