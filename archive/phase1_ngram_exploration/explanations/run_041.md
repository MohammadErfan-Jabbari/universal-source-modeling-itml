# Run 041 — recent-window mixture with window 16384

## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_mix_win16384.py`. It keeps the Run 038 recent-mixture architecture and max recent weight `0.05`, but doubles the online-only recent window:

```text
q = (1 - w_t) q_train+online + w_t q_recent_window
w_t = 0.05 * min(1, past_tokens / 10000)
window = 16384 observed symbols
```

The main and recent components still use n=5 fixed/adaptive interpolation with `alpha=0.3`.

## Source and evidence
- source: experiment observation — Run 038 improved to `2.9740425033` with a 5% recent-window mixture and window `8192`.
- source: experiment observation — Runs 039 and 040 bracketed recent weight: `0.08` and `0.03` were both worse, so keep `0.05` and vary window size instead.
- source: experiment observation — Run 039 suggests too much recent influence overfits local noise; a larger window may reduce variance while preserving recency.
- source: course material — `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` says history recurrence can reveal structure; `lectures/5_04022026_Inequalities_study.md` ties log-loss to model mismatch; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates mixture distributions for robust universal coding.

## Course-material connection
The recent model estimates a local source from synchronized past history. Increasing the window length trades locality for lower estimation variance. This is a bias-variance test within a valid sequential mixture: no future symbols enter the recent counts.

## Hypothesis
If the `8192` recent window is too noisy, a `16384` window will improve by stabilizing recent context estimates. If useful structure changes faster than that, the larger window will dilute local regimes and worsen.

## Risks
This is a public-practice window-size bracket, so avoid fine tuning unless the effect is large. Runtime and memory should remain safe, though the sliding table may retain more contexts. The predictor remains strictly sequential.
