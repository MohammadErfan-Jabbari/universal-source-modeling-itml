## Proposed change
Add `submissions/ngram_interpolated_n5_laplace03_recent_n3_recenta001_w14_warm20000_win262144.py`. It keeps the current best Run 067 architecture and changes only the all-past online component's smoothing:

```text
main model:          n=5, alpha=0.3, train+online counts
```

The main model remains at `alpha=0.3`. This is a very cautious lower-side bracket after `alpha=0.02` still improved but with shrinking gain.

## Source and evidence
- source: experiment observation — Runs 064-067 show monotone improvement as auxiliary smoothing decreases from `0.2` to `0.02`, but the incremental gain is shrinking.
- source: experiment observation — Runs 052-055 showed the all-past low-order component becomes stable with full-prefix history, making low smoothing plausible.
- source: experiment observation — Run 033 showed globally sharp smoothing can hurt sparse `n=5` contexts, so this run leaves the main high-order model unchanged.
- source: official/project rules — `AGENTS.md` and `COMPETITION_RULES.md` require online prediction and fix `N=200000`; all probabilities are computed from past counts only.
- source: course material — `lectures/5_04022026_Inequalities_study.md` ties excess bits to cross-entropy/KL mismatch; `lectures/13_10032026_UniversalCompressionI_LempelZiv_study.md` treats synchronized past history as empirical evidence; `lectures/15_17032026_UniversalCompressionIII_MinimaxRedundancy_study.md` motivates robust mixtures that hedge empirical estimates.

## Course-material connection
A smaller pseudocount makes the auxiliary distribution closer to the empirical online low-order source. That can reduce mismatch when history is dense, but at some point unseen-symbol probability becomes too small and redundancy rises. The main model mixture remains a safety hedge, preserving a valid sequential code.

## Hypothesis
If auxiliary `alpha=0.02` is still over-smoothed, `alpha=0.01` will improve over Run 067 (`2.9718090910`). If the rare-symbol protection limit is around `0.02`, `0.01` will over-sharpen and worsen.

## Risks
This is an aggressive pseudocount and a public-practice calibration run. Rare/unseen continuations may be underweighted, especially early despite the warmup. Runtime and memory should be unchanged.
