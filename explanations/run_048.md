Add `submissions/extra_strong_disagreement_boost_ppma_stack.py`. It keeps the confirmed Run 046 architecture and only increases the JS disagreement multiplier:

## Source and evidence
Run 043 penalized PPM-A/base disagreement and regressed. Run 044 boosted disagreement and improved; Run 046 doubled the boost and improved again, and Run 047 confirmed it exactly. This monotone evidence suggests the current gate may still underweight PPM-A when it carries non-redundant rare-context signal.

Course evidence: Lecture 4's conditional mutual information section says the value of a new signal is its extra predictive value after conditioning on the baseline. The chain rule explicitly decomposes total information into baseline information plus an incremental term. Here JS disagreement is only a proxy, but the empirical sign test says it may track the PPM-A expert's incremental value. Lecture 4's Jensen notes also justify reasoning about mixture/disagreement quantities through convexity.

## Course-material connection
The predictor remains strictly sequential: distributions are functions of train data and the already observed public prefix, and updates happen only after scoring. This is an expert-mixture gate calibration, not a new data source or evaluator change.

## Hypothesis
If disagreement continues to identify useful conditional PPM-A information, the extra-strong boost should improve over Run 046. If Run 046 is near the safe limit, this will over-amplify mismatched contexts and regress, especially because prior train/public transfer has been fragile.

## Risks
This is now close to a gate-strength saturation test, so overfitting risk is real. Keep only if the official score improves and then confirm immediately.
