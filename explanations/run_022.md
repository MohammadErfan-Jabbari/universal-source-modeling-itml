# Run 022 — Selective online updates for probabilistic-depth VOMM

## Proposed change
Add `submissions/selective_vomm_ppma_hybrid.py`. It keeps the Run 010 count-dependent CTW/PPM-C base and the Run 019 selective PPM-A top expert, but changes the Run 014 probabilistic-depth VOMM component: train fitting still fills all suffix depths, while public-practice online updates affect only depth 0 and the deepest available context.

This is not a depth or smoothing sweep. It tests a different online memory policy for a VOMM expert that mixes depths probabilistically.

## Source and evidence
Run 019 showed that selective updates helped PPM-A, while Run 021 showed the same policy slightly hurt PPM-C. That makes the lesson component-specific rather than universal. The next natural test is VOMM: because probabilistic-depth VOMM mixes all depths, noisy intermediate online counts might distort the depth mixture. `autoresearch.ideas.md` lists online adaptation for tree models as a cross-cutting idea and specifically warns that too-aggressive online updates can hurt.

Course evidence: Lecture 4 frames score changes as changes in mismatch redundancy via `H(P,Q)=H(P)+D(P||Q)`, and its conditional-information discussion asks whether a new signal adds predictive value after the context already known. Lecture 14's universal-compression material emphasizes synchronized history as evidence but also the cost/robustness tradeoff of how much history to use.

## Course-material connection
The predictor remains strictly sequential: every evaluation-time update happens only after the symbol has been scored. Selective VOMM updates ask how the online source evidence should enter a variable-order model: should all intermediate contexts learn from each new symbol, or should the model preserve train-derived intermediate statistics and adapt only global and most-specific memories?

## Hypothesis
If intermediate-depth VOMM updates add noise, selective VOMM should beat Run 019. If the VOMM improvement from Run 014 depends on smooth online adaptation across all depths, this will regress. Since VOMM's gate is larger than PPM-A's, a small update-policy effect could matter.

## Risks
Run 021 showed selective updates do not automatically help every tree component. Changing VOMM update policy may also interact with the CTW base and PPM-A top expert in a near-zero-score regime, so any gain needs confirmation.
