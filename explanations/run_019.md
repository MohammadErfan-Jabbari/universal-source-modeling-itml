# Run 019 — PPM-A with selective online updates

## Proposed change
Add `submissions/ppma_selective_update_hybrid.py`, a PPM-A variant mixed into the confirmed Run 014 base. It keeps the same order-5 PPM-A escape law and outer gate as Run 016, but changes the online update architecture: train fitting still populates all suffix orders, while evaluation-time updates add only order 0 and the deepest available context for the PPM-A component. This tests whether intermediate-order online updates add noise or useful adaptation.

## Source and evidence
Run 016 showed that order-5 PPM-A is complementary and Run 017 confirmed it. Run 018 showed that simply reducing PPM-A to order 4 loses the useful signal, so the next PPM-A question is not depth but update semantics. `autoresearch.ideas.md` explicitly lists update rule variation for PPM: update all contexts vs only matched context.

Course evidence: Lecture 13 motivates universal compression through using already observed history as evidence, but the evidence can be noisy when the observed suffix is short. Lecture 4's cross-entropy decomposition says bad online adaptation increases mismatch cost. Selective updates are a robustness mechanism: preserve train-learned intermediate contexts while still adapting the global and most-specific contexts online.

## Course-material connection
The model remains strictly sequential: no future symbols are used, and public-practice symbols affect PPM-A only after prediction. The change is a different universal-coding update policy for how the model's memory is synchronized with the observed stream.

## Hypothesis
If intermediate-order online PPM-A updates were adding noise, selective updates should improve or match Run 016. If all-order updates were necessary for the PPM-A gain, this will worsen. Runtime should remain similar, but the architectural lesson is about online adaptation, not speed.

## Risks
The component may become under-adapted at intermediate depths. The effect is likely tiny and needs confirmation if kept. Runtime remains high because prediction recursion is unchanged.
