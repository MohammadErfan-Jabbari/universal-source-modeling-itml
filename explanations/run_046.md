## Proposed change
Add `submissions/strong_disagreement_boost_ppma_stack.py`. It keeps the confirmed Run 044 architecture but increases the JS disagreement boost:

```text
Run 044: w = w_base * (1 + 0.08 * JS), cap 1.12
```

No expert counts, escape law, or update policy changes.

## Source and evidence
Run 043 penalized PPM-A/base disagreement and regressed. Run 044 boosted disagreement and improved, then Run 045 confirmed the score exactly. That sign test suggests disagreement is a useful proxy for conditional information: the PPM-A expert helps when it differs from the CTW/VOMM/PPM-C base.

Course evidence: Lecture 4's conditional mutual information and chain-rule sections say an added predictor is valuable only by the uncertainty reduction it contributes after the baseline is known. This run asks whether the Run 044 boost was underweighting that incremental term. Lecture 4's Jensen section also motivates using a JS-style disagreement statistic derived from convex/mixture comparisons.

## Course-material connection
The predictor remains strictly sequential and unchanged except for a deterministic gate computed from current past-derived probability distributions. Public symbols are still incorporated only after scoring.

## Hypothesis
If PPM-A/base disagreement is usually useful conditional signal, a stronger boost should improve over Run 044. If Run 044 is near the optimal hedge and further boost amplifies mismatched rare-context overconfidence, this will regress.

## Risks
Gains are tiny and train/public transfer has been fragile. Keep only if primary metric improves and confirm. This is a gate calibration, not a new data source or evaluator change.
