# Run 026 — Order-6 selective PPM-A on confirmed selective tree stack

## Proposed change
Add `submissions/order6_ppma_selective_stack.py`. It keeps the confirmed Run 024 base: unchanged PPM-C/Phase1, selective-update CTW, and selective-update probabilistic VOMM. The only change is the top PPM-A expert: use order 6 instead of order 5 while preserving the selective online update policy and the same conservative support/entropy gate.

## Source and evidence
Run 018 showed order-4 PPM-A was too shallow, while Runs 019 and 024 confirmed that selective online updates make the tree stack more robust. `autoresearch.ideas.md` lists bounded PPM orders 5-7 as a Phase 2 direction. Now that intermediate online-update noise is reduced, order 6 may be safer than it would have been in the all-order-update setting.

Course evidence: Lecture 13 motivates using recurrence as empirical evidence about probability, but longer matches are only useful when they recur often enough. Lecture 4's cross-entropy/KL decomposition says a deeper context helps only if it reduces mismatch; otherwise it adds redundancy. This run tests whether one extra PPM-A context symbol has positive conditional information after the CTW/VOMM base.

## Course-material connection
PPM-A is a sequential universal-coding style model: it uses train counts and the already-decoded public prefix, reserving escape mass for unseen continuations. Increasing the PPM-A order changes the maximum recurrence pattern length the code can exploit, while preserving strict no-lookahead semantics.

## Hypothesis
Order 5 carried useful signal relative to order 4. With selective updates reducing online noise, order 6 may capture additional recurring contexts and improve bits/symbol. If order-6 contexts are too sparse or mismatched, the gate should limit harm, but the score may still regress slightly.

## Risks
This is a bounded-depth PPM-A variant and gains may be tiny. Runtime and memory may rise. The run could overtrust train-only long contexts despite selective public updates, so train-val and stability must be checked.
