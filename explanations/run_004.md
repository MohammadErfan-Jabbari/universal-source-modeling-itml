# Run 004 — Confidence-gated PPM-C + Phase 1 hybrid

## Proposed change
Add `submissions/ppm_phase1_hybrid.py`, a hybrid predictor that keeps the Phase 1 champion as the dominant baseline and adds a substantive PPM-C-with-exclusions component. The PPM component preloads train suffix-trie counts up to order 5 and updates online. Its probability vector is mixed into the Phase 1 vector only when the matched PPM context is well-supported and the PPM distribution has low entropy; otherwise the weight shrinks toward zero. This is not another n-gram sweep: the new signal is an explicit PPM escape/exclusion tree model, but hedged because pure PPM badly mismatched public practice.

## Source and evidence
This follows the cross-cutting Phase 2 idea in `autoresearch.ideas.md`: “PPM + Phase 1 best mixture.” It is motivated by the two preceding PPM runs: no-exclusion PPM-C scored `3.391532`, and exclusion PPM-C improved to `3.240231` but remained far worse than the Phase 1 baseline. Both were deterministic and fast; the durable lesson is that pure PPM has useful train-context structure but overtrusts train-derived contexts on public practice. A confidence-gated hybrid tests whether that structure is complementary when it is only allowed to contribute in high-confidence cases.

Course evidence: Lecture 4 gives `H(P,Q)=H(P)+D(P||Q)`, so a bad pure PPM score means large redundancy from model mismatch, not necessarily that every PPM local prediction is useless. Lecture 13’s universal-compression viewpoint says repeated contexts in the already observed sequence are evidence about probabilities; the hybrid uses that evidence conservatively rather than replacing the stronger learned baseline.

## Course-material connection
The Phase 1 model and the PPM model are two sequential conditional distributions over the next symbol. Mixing them remains a valid online probability model because both see only the past and both update after observation. Information-theoretically, the mixture is a hedge against source uncertainty: if PPM’s recurrence-based distribution is locally informative, it can reduce cross-entropy; if it is mismatched, the Phase 1 model retains most mass and limits KL redundancy.

## Hypothesis
Pure PPM failed because it was too confident under train/public mismatch. A small confidence-gated mixture may still improve contexts where PPM has genuinely low conditional entropy and strong support. The expected gain is small, but this run directly tests whether Phase 2 PPM provides complementary evidence over the Phase 1 champion without letting it dominate.

## Risks
The PPM component may be anti-correlated with public-practice outcomes even in confident contexts, causing the hybrid to be slightly worse than Phase 1. The confidence gate introduces heuristic thresholds, but they are architectural safeguards against the observed PPM overconfidence rather than a sweep of n-gram smoothing constants. Runtime roughly combines Phase 1 and PPM and should remain far below 600 seconds.
