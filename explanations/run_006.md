# Run 006 — Approximate CTW tree mixture + current best hybrid

## Proposed change
Add `submissions/ctw_phase1_hybrid.py`, an approximate Context Tree Weighting (CTW) component mixed into the current best Run 004 PPM+Phase1 hybrid. The CTW component maintains suffix-context continuation counts to depth 5, uses a KT estimator at each node, and recursively blends shorter-context and local estimates with canonical `0.5/0.5` weighting. Unlike Run 005’s min-count VOMM, no single depth is selected; all supported suffix depths contribute through a tree-style recursive mixture. The CTW vector is confidence-gated by support and entropy before being mixed into the base predictor.

## Source and evidence
This follows Direction 2a/2c in `autoresearch.ideas.md`: simplified CTW with KT estimator and a CTW + strong-baseline hybrid. Runs 004 and 005 showed that tree-family additions can produce tiny improvements over the original Phase 1 model, but hard VOMM depth selection was weaker than the PPM escape hybrid. CTW tests a third architecture: model uncertainty over context depth by mixing tree nodes rather than escaping unseen symbols or choosing the deepest supported node.

Course evidence: Lecture 14 explains universal compression through redundancy: an unknown-source code should compete with the code that knew the source, with overhead/redundancy from not knowing the best model. Lecture 4’s `H(P,Q)=H(P)+D(P||Q)` gives the score interpretation: a tree mixture can reduce mismatch if it hedges over context depths better than one fixed or hard-selected depth.

## Course-material connection
CTW is a universal-coding style hedge over context trees. In this implementation, each node’s KT estimate is a valid sequential distribution learned from past symbols, and the recursive 0.5 mixture remains a normalized online probability model. The idea matches the course’s universal-compression theme: when the true source structure is unknown, mix a family of plausible context models to limit redundancy rather than committing too hard to one context length.

## Hypothesis
Phase 1 already has a strong fixed-order interpolation, and Run 004 adds a small PPM escape signal. CTW may add a different signal by smoothing uncertainty over context depth with KT node estimates. If CTW captures contexts where the PPM gate is too sparse or the VOMM hard selector is too brittle, a very small gated weight could improve on Run 004.

## Risks
This approximate CTW is not full alphabet-16 CTW over all child subtrees; it is a suffix-depth recursive mixture, so the theoretical guarantee is weaker. The CTW component may duplicate Phase 1 interpolation and add only runtime. Because Run 004’s gain is tiny and noisy, any further improvement may also be within noise.
