# Run 014 — Probabilistic-depth VOMM + current best

## Proposed change
Add `submissions/prob_depth_vomm_countmix_hybrid.py`, which keeps the current best Run 010 count-dependent CTW hybrid as the base and adds a probabilistic-depth Variable-Order Markov Model. The VOMM stores KT-smoothed local continuation distributions for suffix depths 0..6. Instead of hard-selecting one deepest context (Run 005) or recursively mixing nodes CTW-style (Run 010), it assigns each depth a probability proportional to support, sharpness, and depth, then mixes the local depth distributions. The resulting VOMM expert is added through a small support/entropy gate.

## Source and evidence
This follows `autoresearch.ideas.md` Direction 1c: probabilistic depth VOMM. Prior evidence says hard min-count VOMM was valid but weaker than PPM/CTW hybrids (Run 005), while count-dependent CTW depth 5 is the current best (Run 010/011) and depth-only CTW changes no longer help (Runs 12/13). A probabilistic-depth VOMM is a genuinely different tree-family architecture that may capture depth uncertainty without CTW’s recursive node semantics.

Course evidence: Lecture 4’s cross-entropy identity `H(P,Q)=H(P)+D(P||Q)` says the score improves when the coding distribution reduces mismatch. Variable depth is a way to reduce mismatch across contexts with different relevant memory lengths: high-confidence deeper contexts can reduce conditional uncertainty, while uncertain contexts stay closer to shorter distributions.

## Course-material connection
The model is strictly sequential: counts come from train plus already observed evaluation symbols, and updates occur only after prediction. Information-theoretically, the depth mixture is a model-uncertainty hedge: rather than committing to a single context length, it averages several plausible conditional distributions with weights derived from empirical support and entropy.

## Hypothesis
Run 005’s hard VOMM may have failed because selecting one depth was brittle. A probabilistic depth mixture may extract the useful parts of VOMM while avoiding abrupt wrong-depth decisions. If it captures patterns not already represented by CTW’s recursive mixture, a small gated contribution could improve beyond Run 010.

## Risks
The VOMM expert may duplicate CTW and add only runtime. The depth weights are heuristic, and current best margins are very small. Because deeper train-preloaded models have sometimes improved train-val while hurting public practice, the gate must remain conservative.
