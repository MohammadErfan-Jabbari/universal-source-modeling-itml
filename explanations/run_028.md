# Run 028 — Order-7 selective PPM-A on confirmed selective tree stack

## Proposed change
Add `submissions/order7_ppma_selective_stack.py`. It keeps the confirmed Run 024 base and the Run 026 architecture, but increases the top selective-update PPM-A expert from order 6 to order 7. The update policy and gate stay unchanged: train fitting fills all PPM-A orders, while public-practice updates affect only order 0 and the deepest available context.

## Source and evidence
Run 026 showed a much larger-than-recent improvement when moving selective PPM-A from order 5 to order 6, and Run 027 confirmed it exactly. `autoresearch.ideas.md` lists bounded PPM orders 5-7 as an intended Phase 2 axis. This run tests whether the longer-recurrence signal continues one more level, or whether order 6 is the practical sweet spot.

Course evidence: Lecture 13 explains the universal-compression intuition that repeated blocks are empirical evidence of high probability because recurrence time proxies probability. Lecture 4's cross-entropy/KL decomposition says a deeper context helps only if it reduces mismatch; otherwise it increases redundancy. Order 7 asks whether an additional context symbol still carries conditional predictive information after the CTW/VOMM/PPM-C base and order-6-style gating.

## Course-material connection
The model remains strictly sequential: no public future symbols are used. It is a PPM-style universal coding distribution with explicit escape mass and synchronized online memory, now allowed to exploit one-symbol-longer recurrences when supported.

## Hypothesis
Since order 6 improved substantially over order 5 after selective updates, order 7 may capture additional recurring structure and further reduce bits/symbol. If order-7 contexts are too sparse or train-mismatched, the conservative gate and escape recursion should limit harm, but the primary score may regress.

## Risks
Sparse long contexts can overfit train recurrence and hurt public practice. Runtime and memory may rise. Any gain should be confirmed because Phase 2 improvements are small.
