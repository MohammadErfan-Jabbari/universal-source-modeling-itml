# Run 054 — Support-conditioned disagreement-boosted PPM-A gate

## Proposed change
Add `submissions/support_conditioned_disagreement_boost_ppma_stack.py`. It keeps the confirmed Run 052 experts and escape law, but changes the disagreement boost from a pure scalar to a support-conditioned gate:

```text
Run 052: multiplier = 1 + 0.40 * JS, cap 1.44
Run 054: multiplier = 1 + 0.52 * JS * sqrt(count_conf), cap 1.44
```

where `count_conf = support / (support + 120)`, the same support evidence already used by the outer PPM-A gate.

## Source and evidence
Runs 044, 046, 048, and 050/052 showed monotone improvement from increasing JS disagreement boost, and confirmation runs reproduced the kept scores. However, unbounded scalar boosting risks eventually amplifying low-support mismatches. This run tests whether disagreement should be amplified more when the PPM-A context has stronger support and less when it is sparse.

Course evidence: Lecture 4's conditional mutual information section says an added predictor is useful by the uncertainty reduction left after conditioning on the baseline. The chain rule frames this as incremental information beyond the base stack. Support is a proxy for whether PPM-A disagreement is reliable incremental evidence or noisy mismatch; Lecture 13's recurrence intuition also says repeated contexts are stronger probability evidence than weak recurrence.

## Course-material connection
The predictor remains strictly sequential: train counts and past public symbols only, updates after scoring, no evaluator changes. This is deterministic expert-gate arithmetic using past-derived support and probability distributions.

## Hypothesis
If low-support disagreement is the main overconfidence risk, support-conditioned boosting should improve over Run 052 or at least preserve the trend with better robustness. If useful rare-context signal comes precisely from low-support disagreement, this will regress.

## Risks
Support-gating order 7 once removed useful rare-context signal, so this could be too conservative. Keep only if official primary metric improves and confirm.
