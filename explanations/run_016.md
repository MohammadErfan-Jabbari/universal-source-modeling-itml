# Run 016 — PPM-A escape component + current best

## Proposed change
Add `submissions/ppma_prob_vomm_hybrid.py`, which keeps the current best Run 014 probabilistic-depth VOMM/count-dependent CTW hybrid as the base and adds a small PPM-A-style escape component. The new component is a suffix trie to order 5 with exclusions. At each context it assigns observed continuations `count/(N+1)` and sends a single escape pseudo-count `1/(N+1)` to shorter contexts. This is a different explicit escape law from prior PPM-C and PPM-D attempts, and it is mixed in only through a small support/entropy gate.

## Source and evidence
This follows the Phase 2 PPM direction in `autoresearch.ideas.md`, especially comparing escape methods. Earlier PPM evidence was mixed: pure PPM-C failed, PPM-C as a small gated component helped Run 004, while PPM-D added to the CTW stack hurt public score. Run 014 then showed that a probabilistic tree-depth model can add small deterministic signal. PPM-A tests whether a very conservative escape mechanism adds any complementary zero-frequency handling beyond the current CTW/VOMM stack.

Course evidence: Lecture 14 frames mismatch as extra expected bits; PPM escape probabilities are one way to control mismatch when high-order contexts have not seen a symbol. Lecture 4’s cross-entropy decomposition `H(P,Q)=H(P)+D(P||Q)` is the score lens: better escape calibration should reduce the KL redundancy part of empirical codelength.

## Course-material connection
PPM-A is a valid sequential universal-coding model: predictions use only train plus past evaluation symbols, and update happens after observing the symbol. It explicitly models uncertainty about unseen continuations by reserving escape mass instead of assigning fixed smoothing mass at every depth.

## Hypothesis
PPM-D was too confident and hurt public practice; PPM-C helped only when strongly gated. PPM-A uses a single escape count independent of the number of distinct continuations, so it may be a safer complementary escape signal in high-support contexts. If its errors are not already covered by CTW/VOMM, a small gate could improve bits/symbol.

## Risks
PPM-A may still overfit train contexts or duplicate the existing PPM-C component from Run 004. It also increases runtime on an already slow model. Any gain is likely tiny and should be confirmed if kept.
