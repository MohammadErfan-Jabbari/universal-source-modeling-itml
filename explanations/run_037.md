# Run 037 — Order-7 PPM-A with count-dependent escape

## Proposed change
Add `submissions/order7_count_escape_ppma_stack.py`. It keeps the confirmed order-7 half-escape stack, but makes the escape pseudo-count slightly support-dependent: `escape = 0.5 + 0.25/(N+1)`, where `N` is the usable count in the current context after exclusions. Thus high-support contexts approach the confirmed half-escape law, while very low-support contexts reserve a little more mass for shorter-context uncertainty.

## Source and evidence
Run 033/034 confirmed half escape as best. Run 035 showed quarter escape over-sharpens and hurts public practice despite better train-val; Run 036 showed 0.75 escape is safer but still worse than half. A count-dependent escape centered near 0.5 tests a structural compromise rather than another scalar step: only sparse contexts become more conservative.

Course evidence: Lecture 13 says recurrence is evidence for probability, but weak recurrence is weaker evidence. Lecture 4's `H(P,Q)=H(P)+D(P||Q)` says escape calibration matters because overconfident sparse contexts add mismatch redundancy. This run makes escape mass depend on evidence strength.

## Course-material connection
The predictor remains strictly sequential: train counts plus past public symbols only, PPM exclusions/backoff, and update after scoring. The new rule changes the model's uncertainty allocation across context depths, not evaluator semantics.

## Hypothesis
If half escape is best for supported contexts but sparse contexts still need slightly more backoff, count-dependent escape may beat Run 033. If low-support order-7 contexts are exactly where useful public signal lives, the extra backoff will regress.

## Risks
The effect is likely small. Prior support-gating at order 7 removed useful rare-context signal, so even mild extra escape on sparse contexts may hurt. Any improvement must be confirmed.
