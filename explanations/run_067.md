## Source and evidence
Runs 054, 057, 060, 062, and 064 formed a monotone sequence of small public-practice improvements from square-root support-conditioned JS disagreement boosting, and Run 065 confirmed Run 064 exactly. Because the increments are now tiny, this run uses a half-step (`+0.04` instead of the earlier `+0.08`) to probe saturation more cautiously rather than chasing a large scalar jump.

Course evidence: Lecture 4's conditional mutual information and chain-rule sections frame the PPM-A expert as useful only by its incremental information after the base stack is known. JS disagreement is a proxy for non-redundant conditional signal, while `sqrt(count_conf)` calibrates that signal by recurrence support.

## Course-material connection
The predictor remains strictly sequential: train counts and past public symbols only, with updates after scoring. This is deterministic expert-gate arithmetic and preserves the official evaluator semantics.

## Hypothesis
If Run 064 still slightly underweights reliable square-root-supported PPM-A disagreement, boost `0.88` should improve public log-loss. If the gate is at saturation, this smaller step should reveal regression with less over-amplification than a full `0.92` step.

## Risks
This is close to public-practice calibration and may overfit tiny benchmark differences. Keep only on official primary improvement with stable validation; otherwise roll back and consider diagnostic cap-only or runtime-consolidation experiments.
