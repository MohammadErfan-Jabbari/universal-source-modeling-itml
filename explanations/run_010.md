# Run 010 — Count-dependent CTW + Run 004 hybrid

## Proposed change
Add `submissions/ctw_countmix_phase1_hybrid.py`, a count-dependent CTW-style tree mixture. Like Run 006, it uses a suffix tree to depth 5 and KT estimates at each node. Unlike Run 006’s fixed `0.5/0.5` local-vs-shorter mixture, each node uses `lambda = N/(N+C)` so sparse contexts mostly inherit the shorter-context distribution and high-count contexts can contribute more strongly. The resulting CTW-family distribution is gated into the Run 004 PPM+Phase1 hybrid.

## Source and evidence
This follows `autoresearch.ideas.md` Direction 2b: count-dependent CTW. Run 006’s fixed-half CTW produced the current best and Run 008 confirmed it exactly, while Run 009 showed that simply reducing depth from 5 to 4 lost some of the CTW signal. This run keeps depth 5 but changes the tree-mixture architecture: instead of a canonical fixed hedge at every node, it makes the hedge depend on empirical support.

Course evidence: Lecture 14’s minimax redundancy section explains coding with an unknown source as minimizing mismatch penalty. A count-dependent tree mixture is a robustness mechanism: avoid paying KL/codelength redundancy for sparse, unreliable contexts while still exploiting contexts whose recurrence gives strong evidence. Lecture 4’s `H(P,Q)=H(P)+D(P||Q)` provides the score interpretation.

## Course-material connection
The model is still strictly sequential: every CTW node count is fitted from train and then updated only after public-practice symbols are observed. Count-dependent CTW is a universal-coding hedge over context depths where the amount of trust in a deeper context grows with empirical evidence. This directly implements the bias-variance tradeoff behind redundancy control: unsupported detail can hurt, but well-supported detail can reduce conditional uncertainty.

## Hypothesis
Run 006 proved depth-5 CTW has a small reproducible signal. Fixed 0.5 mixing may underweight high-count informative contexts and overweight sparse contexts. Count-dependent mixing may preserve the useful depth-5 signal while better calibrating sparse contexts, potentially improving public-practice bits/symbol beyond Run 006.

## Risks
This may behave too much like an interpolated n-gram if the count-dependent weights dominate, but the architecture remains a recursive CTW/tree model rather than a fixed-order smoothing sweep. The chosen support constant is heuristic, and the score differences are tiny; any result must be interpreted cautiously against Run 006 and train-val.
