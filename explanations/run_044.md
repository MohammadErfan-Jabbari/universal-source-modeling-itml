# Run 044 — Disagreement-boosted PPM-A mixture weight

## Proposed change
Add `submissions/disagreement_boost_ppma_stack.py`. It keeps the Run 037 experts and count-dependent PPM-A escape law unchanged, but makes the final PPM-A gate slightly larger when PPM-A differs from the base:

```text
w = w_run037 * (1 + 0.08 * JS(base, PPM-A))
```

with the multiplier capped at `1.12`. This is the opposite test of Run 043, which penalized JS disagreement and regressed.

## Source and evidence
Run 043 taught that suppressing PPM-A/base disagreement worsens both public and train-val. That implies PPM-A's value may be conditional information: useful signal not already present in the CTW/VOMM/PPM-C base. Runs 039-042 suggest adding new escape tweaks or direct LZ recurrence is saturated, so this run changes only the expert gate.

Course evidence: Lecture 4's conditional mutual information section says the value of a new signal is the uncertainty reduction left after conditioning on a baseline. Its chain rule describes the extra information term. Here the base stack is the baseline, and disagreement is a crude proxy for PPM-A carrying non-redundant information. Lecture 13's recurrence view still motivates PPM-A as a recurrence-derived expert.

## Course-material connection
The predictor remains strictly sequential: all distributions are computed from train and past public symbols, and all public updates occur after scoring. This is an expert-mixture question: should a PPM-style recurrence expert get slightly more probability mass when it brings evidence the base does not already encode?

## Hypothesis
If Run 043 regressed because disagreement was useful rare-context signal, a small disagreement boost may beat Run 037. If disagreement is a mix of signal and mismatch already optimally weighted by Run 037, this will regress.

## Risks
This is a very small gate change and may be noise-scale; keep only if it improves and confirm. Boosting disagreement can amplify mismatched overconfidence, especially given prior train/public transfer warnings.
