# Run 033 — Order-7 PPM-A with half escape pseudo-count

## Proposed change
Add `submissions/order7_half_escape_ppma_stack.py`. It keeps the confirmed order-7 selective PPM-A stack, but changes the PPM-A escape law from a full single pseudo-count to a half pseudo-count: observed continuations get `count/(N+0.5)` and the recursive escape gets `0.5/(N+0.5)`. The selective online update policy and outer support/entropy gate remain unchanged.

## Source and evidence
Run 028/029 confirmed order 7 as the public-practice sweet spot. Run 030 showed order 8 overfits/mismatches public practice, and Run 032 showed support-gating order 7 removes useful rare-context signal. So the next non-depth question is escape calibration: can the useful order-7 rare contexts be trusted slightly more without going deeper?

Course evidence: Lecture 13 treats recurrence as evidence for probability; escape mass controls how much probability is reserved for shorter-context uncertainty when a continuation was unseen at the current depth. Lecture 4's `H(P,Q)=H(P)+D(P||Q)` says better escape calibration reduces mismatch redundancy; too much or too little escape mass can add bits.

## Course-material connection
This remains a valid sequential PPM-style universal code: it uses train plus past public symbols only, excludes seen symbols during backoff, and updates after prediction. The change is the escape distribution over model uncertainty, not any access to future data.

## Hypothesis
If order-7 contexts are useful but the full PPM-A escape pseudo-count is too conservative, half escape may sharpen the PPM-A expert and improve bits/symbol. If the current escape mass is already protecting against sparse-context mismatch, this will regress.

## Risks
This is an escape-law calibration and may be a small effect. Sharpening can overfit rare contexts. Any improvement should be confirmed.
